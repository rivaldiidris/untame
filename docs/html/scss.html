SCSS (SASS-CSS)

We choose SASS as our preprocessor CSS language. SASS lets you use features that don't exist in CSS yet like variables, nesting, mixins, inheritance and other nifty goodies that make writing CSS fun.

Installing SASS

Before you start using Sass you will need to install Ruby. Download Ruby

After finished installing Ruby, open your Command Prompt and type below code:

gem install sass

Compiler

We need compiler to generate the CSS file. One of recommended free application is Koala. Download Koala

Add project

  1. Drag and drop a folder into the window to add a project, or click on the plus icon in the top left corner and select your project folder.
    Select main folder of each project, Koala will automatically searching for .scss files and listed it.
  2. Click the SCSS file options menu will appear from right, default option of Auto Compile is ticked.
    Any changes of SCSS file from other editor Koala will automatically compile the file.

WebStorm IDE Setup

You must install the Ruby as stated above before configuring this.

We can also use WebStorm editor to compile our SCSS file to CSS with some configuration

  1. Open or create your project folder in WebStorm then click settings
  2. Select Tools > File Watcher
  3. Add a new file watcher by clicking + in top right and select SCSS
  4. By default the option Immediate file synchronization is ticked, you can untick this option if you don't need it auto compile immediately after you finish typing

Please write your SCSS code based on the partial files, and put it all inside of part folder with _/underscore prefix of naming file.

  1. style.scss In this file please put only Fonts and Import Files
  2. _base.scss In this file put your style about base element properties. ie.body, margin, padding, p, H1, H2 etc.
  3. _custom-bootstrap.scss If you want to edit Bootstrap style, please put it here.
    Please note to avoid using !important, instead please use specific class parent.
  4. _custom-plugins.scss If we have external plugins CSS from third party, please do custom style here. Please note to not edit the default CSS plugins and avoid using !important
  5. _fonts.scss Global fonts (e.g. embedded or external fonts) definition put it all here
  6. _global-style.scss Global element properties and classes put it all here
  7. _layout.scss Main layout header, content, nav, aside, footer styling are here
  8. _mixins.scss Collect groups of CSS declarations that you want to reuse throughout your site here
  9. _modules.scss Group of widgets, modules like social media, tabs, slider etc.
  10. _variables.scssCollect all variables that you want to reuse throughout your stylesheet like group of colors here
